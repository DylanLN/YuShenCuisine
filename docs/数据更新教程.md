# 🗺️ 数据更新教程

## 快速更新（一键）

```bash
bash scripts/sync-data.sh
```

这条命令会自动完成：
1. **调用高德 API** 搜索 14 个关键词 × 10 个区
2. **按坐标过滤**（只留深圳范围内的店）
3. **重新分配区名**（按坐标判断属于哪个区）
4. **去重**（重复的只保留一条）
5. **输出到前端**（自动复制到 web 目录）
6. **提交 git**

### 注意事项

- 每次更新前先确认 `crawler/config.py` 里的关键词是否够用
- 如果采集途中报错，一般是高德 API 限流，等 1 分钟重跑就行

## 手动增删改

数据文件位置：`web/src/assets/data/restaurants.json`

### 新增一家店

在 JSON 数组末尾加一条：

```json
{
  "id": "",
  "name": "店名",
  "district": "福田",
  "address": "详细地址",
  "phone": "0755-xxxxxxxx",
  "longitude": 114.05,
  "latitude": 22.55,
  "rating": null,
  "cost": null,
  "tags": ["烩面"],
  "cover": "",
  "source_keyword": "手动添加",
  "update_time": "2026-06-24"
}
```

各字段说明：

| 字段 | 必填 | 说明 |
|------|------|------|
| `id` | 否 | 高德POI ID，手动添加的留空 |
| `name` | 是 | 店名 |
| `district` | 是 | 区名：福田/罗湖/南山/盐田/宝安/龙岗/龙华/坪山/光明/大鹏 |
| `address` | 是 | 地址 |
| `phone` | 否 | 电话，没有就 `""` |
| `longitude` | 是 | 经度（GCJ-02） |
| `latitude` | 是 | 纬度（GCJ-02） |
| `rating` | 否 | 评分 0-5，没有就 `null` |
| `cost` | 否 | 人均消费，没有就 `null` |
| `tags` | 是 | 分类标签：烩面/胡辣汤/灌汤包/羊肉汤/河南菜 |
| `source_keyword` | 否 | 来源关键词，手动添加就写 "手动添加" |
| `update_time` | 是 | 更新日期 |

### 经纬度怎么查？

用高德地图网页版：

1. 打开 https://ditu.amap.com/
2. 搜地址
3. 右键点位置 → 「显示坐标」
4. 复制经纬度

### 修改/删除

直接编辑或删除 `restaurants.json` 里对应的条目。

## 修改搜索关键词

编辑 `crawler/config.py` 里的 `KEYWORDS`：

```python
KEYWORDS = [
    "河南菜", "河南饭店", "河南餐馆", "烩面", "胡辣汤",
    # 想加就加：
    "水煎包", "豫北饭店",
    # 不想要就删
]
```

改完后跑 `bash scripts/sync-data.sh` 重新采集。

---

## 新增分类标签

两步：

1. 在 `crawler/config.py` 的 `TAG_RULES` 里加规则：

```python
TAG_RULES = {
    # ... 原有的 ...
    "水煎包": "水煎包",    # 新增
    "道口烧鸡": "道口烧鸡",  # 新增
}
```

2. 跑 `bash scripts/sync-data.sh` 重新采集

## 文件索引

| 文件 | 用途 |
|------|------|
| `crawler/config.py` | 关键词、行政区、TAG_RULES 配置 |
| `crawler/crawl.py` | 高德 API 采集脚本 |
| `crawler/process.py` | 数据清洗（过滤+去重+分配区名） |
| `scripts/sync-data.sh` | 一键更新脚本 |
| `web/src/assets/data/restaurants.json` | **前端用的数据**（改这里） |
| `web/.env` | 高德 JS API Key（地图展示用） |
